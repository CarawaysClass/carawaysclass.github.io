Semester Project #2 - Interface and Database Design Document

Overview

Building on what you developed in our first semester project (the Systems Requirements document) you will develop the initial User Interface design and the initial Database design.

Learning Activities

  1. Review your System Requirements Document submission.
  2. Revisit the SCR Case Simulation for units 7 - 9 in MindTap.
  3. Review the related unit readings.

Assignment Tasks

  1. From the SCR Case Simulation Session 8:
    1. Create a detail report that will display all SCR courses in alphabetical order, with the course name and the instructor name in a group header; the Social Security number, name, and telephone number of each current student in the detail section; and the student count in a group footer.
    2. Create a switchboard design with control buttons that lead to students, instructors, courses, course schedules, and course rosters. Allow a user to add, update, or delete records in each area. Consider laying these out as a storyboard of your proposed screens.
    3. Within your design, list the appropriate data validation checks for the data entry screens.
  2. From the SCR Case Simulation Session 9:
    1. List all the entities that interact with the TIMS system by reviewing your requirements document including the data library, DFDs, etc.
    2. Draw an ERD that shows cardinality relationships among the entities.
    3. For each entity, make sure your table design is in 3NF. Use standard notation format to show the primary key and the other fields in each table.
    4. For each table, provide sample data for all fields for at least three records.

Grading Rubric

This assignment is worth a total of 200 points. The Grading Rubric breaks out as follows:

  1. Detail report (50 pts)
  2. Switchboard Design (70 pts)
  3. Entity List (20 pts)
  4. Entity Relationship Diagram (60 pts)

Submitting Your Work

  1. Create a single PDF document that includes all of the other components (Detail Report, Switchboard Design, Entity List, and ERD) and upload that in the D2L Dropbox by the Due Date.
